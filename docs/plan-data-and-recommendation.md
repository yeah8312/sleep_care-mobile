# 데이터 및 추천 계획

## 핵심 데이터
### 학습 세션 데이터
- 세션 ID
- 시작/종료 시각
- 연결된 기기 정보
- 세션 상태

### 졸음 이벤트 데이터
- 발생 시각
- 감지 강도 또는 신뢰도
- 지속 시간
- 감지 당시 상태 정보
- 기기 ID

### 심박 데이터
- 샘플 시각
- BPM
- IBI
- 품질 라벨
- 전달 모드

### 수면 데이터
- 수면 시작 시각
- 수면 종료 시각
- 총 수면 시간
- 수면 품질 관련 지표
- 동기화 시각

### 사용자 입력 데이터
- 학습 플랜
- 시험 날짜
- 목표 기상 시간
- 선호 취침 시간대

## 분석 결과 데이터
- 일별 졸음 빈도
- 주간 수면 규칙성
- 수면 부족 가능성 지표
- 권장 취침 및 기상 시간
- 시험 대비 수면 조정 가이드

## 데이터 저장 전략
### Room
- 학습 세션
- 졸음 이벤트
- 심박 요약 또는 캐시
- 수면 세션
- 시험 일정
- 학습 플랜
- 추천 결과 캐시

### DataStore
- 설정값
- 마지막 동기화 시각
- 연결된 기기 정보
- 사용자 선호값

## 초기 추천 로직 방향
- 규칙 기반 분석 우선
- 최근 며칠간 수면 부족 여부 계산
- 졸음 발생 시간대와 수면 시간 상관관계 단순 분석
- 시험 일정과 학습 플랜을 고려한 권장 취침 시간 제안
- 실시간 심박 데이터는 장기 추천의 주 지표보다 세션 해석 보조 지표로 사용

## 처리 구조 기준
- Repository 계층에서 라즈베리파이 졸음 데이터, 워치 세션 데이터, 수면 데이터, 사용자 입력 데이터를 통합한다.
- ViewModel + Coroutines + Flow 조합으로 조회 상태와 추천 결과 상태를 화면에 전달한다.
- 주기적 리포트 계산과 재시도 가능한 동기화 작업은 WorkManager로 분리한다.
- 추천 결과는 캐시해 홈 화면과 상세 화면에서 같은 결과를 재사용할 수 있게 한다.

## 고도화 방향
- 개인화 모델 적용
- 사용자 반응 기반 추천 조정
- 장기 패턴 분석
- 컨디션 예측 기능 검토

## 추천 예시
- 최근 3일 평균 수면이 부족하고 오후 졸음이 자주 발생하면 더 이른 취침 시간을 제안한다.
- 시험 날짜가 가까워질수록 시험 시간에 맞춘 기상 리듬 조정을 제안한다.
- 아침 공부 플랜이 많으면 취침 시간을 더 일찍 제안한다.

## 협업 논의 포인트
- 졸음과 수면의 상관관계 시각화 방식
- 추천 결과 설명 문구 형식
- 심박 데이터의 장기 저장 범위
- 개인화 수준과 기준
- 참고 의견: 초기 추천은 정교한 AI보다 규칙 기반 추천으로 시작해도 충분하다.
- 참고 의견: 추천 결과는 왜 그런 제안이 나왔는지 설명 가능해야 사용자가 신뢰하기 쉽다.
