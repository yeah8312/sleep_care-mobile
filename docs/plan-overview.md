# Sleep Care 프로젝트 개요

## 프로젝트 한 줄 정의
수험생의 공부 중 졸음과 실제 수면 패턴을 함께 분석해 더 좋은 수면 루틴을 제안하는 통합 케어 프로젝트.

## 제품 구성
- 라즈베리파이 디바이스: 카메라 기반 안구 상태 감지, 즉시 경고, 실시간 위험도 계산
- 모바일 앱: 세션 제어, 데이터 허브, 기록 저장, 분석, 수면 스케줄 제안
- 워치 앱: 학습 세션 중 심박/IBI 전달, 진동 보조 경고, 수면 데이터 연동 보조

## 핵심 사용자 가치
- 공부 중 졸음 상태를 실시간으로 파악한다.
- 실제 수면 패턴과 졸음 패턴을 함께 분석한다.
- 학습 플랜과 시험 날짜에 맞춘 수면 스케줄을 제안한다.

## 기본 사용자 흐름
1. 사용자가 모바일 앱에서 프로필과 시험 일정, 학습 플랜을 설정한다.
2. 모바일 앱이 워치와 라즈베리파이를 연결한다.
3. 사용자가 공부 세션을 시작하면 워치는 심박/IBI를 보내고 라즈베리파이는 안구 상태를 감지한다.
4. 모바일 앱은 워치 데이터를 중계하고 세션 로그를 저장한다.
5. 라즈베리파이는 안구 데이터와 심박 데이터를 바탕으로 졸음 위험도를 계산하고 즉시 경고한다.
6. 모바일 앱은 수면 기록과 학습 세션 기록을 함께 분석해 수면 스케줄과 생활 가이드를 제안한다.

## 시스템 역할 분리
- 라즈베리파이: 안구 감지, 실시간 졸음 판단, 현장 경고
- 워치: 학습 세션 중 심박/IBI 수집, 진동 경고, 수면 데이터 소스 연동 보조
- 모바일 앱: 세션 제어, 데이터 중계, 저장, 분석, 추천, 사용자 경험 제공

## 기술 스택 요약
- 앱 본체: Kotlin
- UI: Jetpack Compose + Material 3
- 아키텍처: MVVM + Repository + UDF(단방향 데이터 흐름)
- 상태관리 / 비동기 / DI: ViewModel + Kotlin Coroutines + Flow + Hilt
- 로컬 저장: Room + DataStore
- 백그라운드 작업: WorkManager
- 워치 실시간 통신: Wear OS Data Layer
- 라즈베리파이 통신: 로컬 네트워크(NSD + WSS)
- 수면 데이터 연동: Health Connect 및 플랫폼 API 조합 검토

## 핵심 의사결정 포인트
- 1차 대상 워치 플랫폼과 센서 접근 방식
- 수면 데이터의 실제 접근 경로
- 라즈베리파이 안구 감지 MVP 기준
- 모바일 앱의 세션 제어와 재전송 정책
- 데이터 저장 구조와 추천 엔진 범위

## MVP 범위
- 라즈베리파이 기본 졸음 감지와 현장 경고
- 워치 심박/IBI 실시간 전달과 진동 보조 경고
- 모바일 앱의 세션 로그 저장과 기본 분석
- 수면 데이터 연동과 간단한 수면 스케줄 제안
